**核心结论**(数据采集于 2026-07-11,对象为 [langgenius/dify](https://github.com/langgenius/dify) commit 数前 100 的贡献者,使用开源的 [ghfind 引擎](/methodology)评分):

- **Dify 前 100 贡献者的中位分是 78.2 / 100,比我们 18,947 个账号基线的中位数 42.5 高出 36 分。** 70.8% 越过"可信贡献者"线(≥70 分),而全站人群只有 20.1% 能达到。
- **只看 followers,他们和普通开发者没有区别;看产出,完全是另一回事。** followers 中位数 31(基线 27),合并 PR 中位数 118(基线 20)。写 Dify 的是一群不出名但产出极高的专业开发者,不是你在社交媒体上认识的任何人。
- **没有谁一个人扛着整个项目——这在明星项目里很少见。** commit 最多的人只写了 8.4%,前三名合计 21%,前十名合计不到一半。对比一下[那些基本靠一个人写出来的爆红 AI 项目](/blog/who-builds-openclaw),是完全不同的活法。

## 为什么研究 Dify

[Dify](https://github.com/langgenius/dify) 是 GitHub 上 star 数最高的 LLM 应用平台之一——148,500 个 star、458 名代码贡献者,创建于 2023 年 4 月。star 数是大家判断项目好坏的默认指标,也是最容易注水的指标。所以我们问了一个 star 数回答不了的问题:**这个项目到底是谁在写?这些人的履历经得起验证吗?**

我们取 commit 数前 100 的贡献者,剔除 3 个 bot(`dependabot`、`github-actions`,以及——很有时代感——`Copilot`),对剩下 97 个人类账号跑确定性评分。96 个成功解析,86 个留有完整原始指标快照。引擎与本站所有分数同源:六个维度共 100 分,不调用任何模型,相同输入必得相同分数。

## 贡献者质量:远高于基线

| | Dify 前 100 | 19k 基线 |
|---|---|---|
| 中位分 | **78.2** | 42.5 |
| ≥ 90 分(夯) | **15.6%** | 3.7% |
| ≥ 70 分(可信) | **70.8%** | 20.1% |
| < 40 分(低价值) | **5.2%** | 48.6% |

这两个分布几乎不重叠。Dify 排在第 10 百分位的贡献者(56.1 分)都比全站中位数高 13 分。96 人中有 15 人拿到 90 分以上——放眼整个 GitHub,大约二十七个账号里才有一个能做到。

低于 40 分的那 5 个账号也值得说一句:任何热门仓库都会攒下一批"提交过一次、主页上没什么别的东西"的人。在健康的项目里,这样的人大约占前 100 名的 5%;而在基线人群里,接近一半的账号都在 40 分以下。

## 安静的专业人士

最有意思的发现,是这群人的知名度和产出完全不成比例:

| 每人中位数 | Dify 前 100 | 19k 基线 |
|---|---|---|
| followers | 31 | 27 |
| 合并 PR 数 | **118** | 20 |
| 账号年龄 | 9.0 年 | 7.4 年 |

按 followers 看,Dify 的核心贡献者和 GitHub 上的普通账号没有任何区别;按合并 PR 数看,他们的产出是基线的**六倍**。这正好是我们在 [19k 账号研究](/blog/we-scored-19000-github-accounts)里记录的造假画像的反面——造出来的账号会把人人看得见的数字刷得漂亮,却不干实际的活;Dify 的贡献者是反过来的:活干得很多,名声懒得经营。

账号年龄也印证这一点:86 个有完整数据的账号里,只有 4 个注册不满一年。这不是一群新号在蹭热门仓库——典型的 Dify 贡献者 2017 年就已经在 GitHub 上了。

## 即使在这里,刷出来的账号也存在

前 100 名里有 2 个账号(2.1%)超过了引擎判定"刷贡献"的阈值——所谓刷贡献,就是靠大量低成本、重复性的 PR 造出一份贡献履历。基线人群里这个比例是 0.58%。用当前版本引擎对最新快照重算,2 个中有 1 个仍然命中(1.2%)。各类警示信号出现在 13/86 个账号上(15.1%,基线 17%),但内容几乎全是"这个主页太单薄"(`mostly_forks`:12 个),而不是"这个主页是假的":批量生产 PR 标题的只有 1 个,靠琐碎 PR 堆履历的也只有 1 个。

我们只公布汇总数字,但道理是通用的:**项目再有名,也不能保证前 100 名贡献者个个干净。** 名气恰恰会招来这种行为——往著名仓库里塞几个改动极小的模板 PR,是市面上成本最低的镀金方式。这正是我们正在为维护者开发的工具要抓的东西。

## 很多双手一起写的项目

前 100 样本的 commit 总数:8,434。

| commit 占比 | |
|---|---|
| 最活跃的 1 人 | 8.4% |
| 前 3 名 | 21.0% |
| 前 5 名 | 31.3% |
| 前 10 名 | 49.8% |

超过一半的 commit 来自前十名**之外**。有名的 AI 项目往往压在一两个疲惫的维护者身上,人一停项目就停;Dify 把活摊到这么多人手里,想垮都难。榜单头部混着 LangGenius 的员工和独立的社区维护者,从第 1 名(708 个 commit)到第 10 名(249 个)是平缓的坡,不是悬崖。

## 方法与局限

分数来自 ghfind 的确定性规则——六个维度、只用公开 GitHub 数据、不调用模型,以 AGPL 开源。完整规则与阈值见[方法论](/methodology);每张表背后的汇总数据见 [data.json](/blog/who-builds-dify/data.json)。

- **前 100(按 commit 数)不等于整个社区。** Dify 有 458 名代码贡献者,我们评的是最活跃的那一头;偶尔来提交一次的长尾,分数大概率更低。
- **这是一张快照。** 数据采集于 2026-07-11,star 和 commit 每天都在变;实时贡献者数据见 [Dify 项目页](/developers/repo/langgenius/dify)。
- **commit 数衡量的是数量,不是重要性。** 占比表说的是谁提交得最多,不是谁写了整个项目最依赖的那部分代码。
- **刷量相关的发现只公布汇总数字。** 点名只在说好话的时候——这是本站的固定规矩。
