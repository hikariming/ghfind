**核心结论**(数据采集于 2026-07-11,对象为 [langgenius/dify](https://github.com/langgenius/dify) commit 数前 100 的贡献者,使用开源的 [ghfind 引擎](/methodology)评分):

- **Dify 前 100 贡献者的中位分是 78.2 / 100,比我们 18,947 个账号基线的中位数 42.5 高出 36 分。** 70.8% 越过"可信贡献者"线(≥70 分),而全站人群只有 20.1% 能达到。
- **只看 followers,他们和普通开发者没有区别;看产出,完全是另一回事。** followers 中位数 31(基线 27),合并 PR 中位数 118(基线 20)。Dify 是一群低调、高产的专业开发者写出来的,不是网红写出来的。
- **对一个这么热的项目来说,bus factor 罕见地健康。** 第一名只占样本 commit 的 8.4%,前三名占 21%,前十名占 49.8%——对比爆红 AI 项目常见的["一人军团"模式](/blog/who-builds-openclaw),这是另一个物种。

## 为什么研究 Dify

[Dify](https://github.com/langgenius/dify) 是 GitHub 上 star 数最高的 LLM 应用平台之一——148,500 个 star、458 名代码贡献者,创建于 2023 年 4 月。star 数是衡量项目健康度的默认指标,也是最容易注水的指标。所以我们问了一个 star 数回答不了的问题:**这个项目到底是谁在写?这些人经得起贡献真实性审计吗?**

我们取 commit 数前 100 的贡献者,剔除 3 个 bot(`dependabot`、`github-actions`,以及——时代特色——`Copilot`),对剩下 97 个人类账号跑确定性评分。96 个成功解析,86 个留有完整原始指标快照。引擎与本站所有分数同源:六个维度共 100 分,不调用任何模型,相同输入必得相同分数。

## 贡献者质量:远高于基线

| | Dify 前 100 | 19k 基线 |
|---|---|---|
| 中位分 | **78.2** | 42.5 |
| ≥ 90 分(夯) | **15.6%** | 3.7% |
| ≥ 70 分(可信) | **70.8%** | 20.1% |
| < 40 分(低价值) | **5.2%** | 48.6% |

这个分布与全站人群几乎不重叠。Dify 第 10 百分位的贡献者(56.1 分)都比全站中位数高 13 分。96 人中有 15 人进入最高段位——全站只有二十七分之一的账号能进。

低于 40 分的 5 个账号也值得一提:任何热门仓库都会积累一条"单次提交、主页单薄"的长尾。5% 就是健康项目的长尾长度——作为对照,基线人群有接近一半低于 40 分。

## 安静的专业人士

最有意思的模式,是可见度与产出之间的落差:

| 每人中位数 | Dify 前 100 | 19k 基线 |
|---|---|---|
| followers | 31 | 27 |
| 合并 PR 数 | **118** | 20 |
| 账号年龄 | 9.0 年 | 7.4 年 |

按 followers 看,Dify 的核心贡献者和 GitHub 平均账号没有区别;按合并 PR 量看,他们的产出是基线的**六倍**。这正好是我们在 [19k 账号研究](/blog/we-scored-19000-github-accounts)中记录的刷量画像的反面——刷出来的账号优化可见指标、跳过实际工作,Dify 的贡献者群体优化实际工作、跳过可见指标。

账号年龄分布进一步印证:86 个深度样本账号中只有 4 个不满一年。这不是一群新号在追热门仓库;中位贡献者 2017 年就已经在 GitHub 上了。

## 即使在这里,刷量也存在

前 100 名中有 2 个账号(2.1%)超过我们的刷量模式阈值(内部垃圾分 ≥ 3),基线是 0.58%;用当前版本引擎对最新快照重算,命中 1 个(1.2%)。任意风险信号命中 13/86(15.1%,基线 17%),构成以**缺失型**信号为主(`mostly_forks`:12 个),操纵型极少:模板化 PR 灌水 1 个、微小 PR 刷量 1 个。

我们只做聚合披露,但这个结论可以外推:**明星项目的前 100 贡献者列表并不天然干净。** 热度会吸引模式化刷量——往高星仓库塞小的模板 PR,是市面上成本最低的履历。这正是我们正在为维护者构建的反垃圾工具要解决的问题。

## 一支真正分布式的队伍

前 100 样本的 commit 总数:8,434。

| 集中度 | commit 占比 |
|---|---|
| 第 1 名 | 8.4% |
| 前 3 名 | 21.0% |
| 前 5 名 | 31.3% |
| 前 10 名 | 49.8% |

一半的 commit 量在前十名**之外**。对一个 14.8 万 star 的 AI 项目——一个以"独狼维护者过劳"闻名的品类——这是罕见的抗风险结构。榜单头部混合了 LangGenius 团队成员与独立社区维护者,从第 1 名(708 commits)到第 10 名(249)的衰减是平缓的坡,不是悬崖。

## 方法与局限

分数来自 ghfind 的确定性规则——六个维度、只用公开 GitHub 数据、不调用模型,以 AGPL 开源。完整规则与阈值见[方法论](/methodology);每张表背后的聚合数据见 [data.json](/blog/who-builds-dify/data.json)。

- **前 100(按 commit 数)不等于整个社区。** Dify 有 458 名代码贡献者,我们评的是分布的头部,长尾大概率更薄。
- **时点数据。** 采集于 2026-07-11。star 和 commit 每天都在变,实时贡献者数据见 [Dify 项目页](/developers/repo/langgenius/dify)。
- **commit 数衡量的是数量,不是重要性。** bus factor 按前 100 样本计算,不代表谁写了承重墙代码。
- **刷量数据只做聚合披露。** 点名只用于正面语境,这是本站的固定政策。
