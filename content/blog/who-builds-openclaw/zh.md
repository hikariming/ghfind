**核心结论**(数据采集于 2026-07-11,对象为 [openclaw/openclaw](https://github.com/openclaw/openclaw) commit 数前 100 的贡献者,使用开源的 [ghfind 引擎](/methodology)评分):

- **GitHub 历史上涨星最快的仓库,按 commit 量算基本是一个人写的。** 创始人 [steipete](/u/steipete) 一人贡献了样本 58,487 个 commit 中的 33,482 个——**57.2%**;前三名合计 81.5%,前十名 90.2%。
- **零刷量。** 96 个人类贡献者中,没有任何一个超过我们的刷量模式阈值——而即使在自选择偏差明显的 18,947 账号基线里,这个比例也有 0.58%。热度是天文数字,热度背后的贡献者名单是干净的。
- **AI Agent 浪潮正在把新人卷进开源。** 前 100 贡献者中 19.6% 的账号注册不满一年(作为对照,同样量级的老项目 Dify 只有 4.7%)。其中已经出现顶级选手:commit 数第 3 的贡献者用一个 2.3 年的账号拿到了 94.1 分。

## 为什么研究 OpenClaw

[OpenClaw](https://github.com/openclaw/openclaw) 不需要介绍:2025 年 11 月 24 日由 Peter Steinberger([steipete](/u/steipete),PSPDFKit 创始人)创建,2026 年 1 月一周之内改了两次名(Clawdbot → Moltbot → OpenClaw),是史上最快达到 10 万 star 的项目。截至 2026-07-11,它有 **382,580 个 star、80,292 个 fork、368 名代码贡献者**——距第一个 commit 大约七个半月。

这样的涨星曲线正是我们的引擎为之而生的场景:当一个数字垂直起飞,它下面到底有没有真东西?我们取 commit 数前 100 的贡献者,剔除 4 个 bot(包括项目自己的 `clawsweeper` 和 `openclaw-clownfish`),对全部 96 个人类账号评分,其中 92 个留有完整原始指标快照。

## 贡献者质量:头部是精英

| | OpenClaw 前 100 | Dify 前 100 | 19k 基线 |
|---|---|---|---|
| 中位分 | **79.7** | 78.2 | 42.5 |
| ≥ 90 分(夯) | **21.9%** | 15.6% | 3.7% |
| ≥ 70 分(可信) | **69.8%** | 70.8% | 20.1% |
| < 40 分(低价值) | **9.4%** | 5.2% | 48.6% |

(Dify 列来自[同系列研究](/blog/who-builds-dify),同一周、同一引擎。)

OpenClaw 每五个头部贡献者就有一个进入全站只有二十七分之一账号能进的最高段位。愿意来给年度最热仓库写代码的人,不成比例地优秀——垂直的关注度招募来垂直的人才。但注意另一条尾巴:9.4% 低于 40 分,几乎是 Dify 的两倍。热度磁铁也会吸来单薄的账号;下文的年龄结构会解释这件事。

## 一双手

前 100 样本的 commit 总数:58,487——是 Dify(8,434)的七倍,而日历时间只有五分之一。

| 集中度 | OpenClaw | Dify |
|---|---|---|
| 第 1 名 | **57.2%** | 8.4% |
| 前 3 名 | **81.5%** | 21.0% |
| 前 5 名 | **86.0%** | 31.3% |
| 前 10 名 | **90.2%** | 49.8% |

[steipete](/u/steipete) 的 33,482 个 commit 摊在 229 天里,平均**每天 146 个**。这不是人类的打字速度,这是人类的*审核*速度——OpenClaw 出了名地由它自己的 agent 在创始人的指挥下构建,这份 commit 记录就是"AI 放大的单人开发"全速运转的样子。引擎给这个账号打了 **100 分满分**:17 年 GitHub 历史、52,067 个 followers、2,772 个合并 PR——与机器人模式截然相反。规模是真的,只是它集中在一双手上,集中到任何一个 38 万 star 项目都从未有过的程度。

第二梯队人少但过硬:[vincentkoc](/u/vincentkoc)(10,502 commits,96.5 分)、[shakkernerd](/u/shakkernerd)(3,688,94.1 分)、[obviyus](/u/obviyus)(1,771,93.2 分)。第 10 名之后,再没有任何人占到 commit 总量的 0.5%。

两种结构都能出货:Dify 运转着一个真正分布式的贡献者经济,OpenClaw 运转着一台前所未有节奏的开明独裁机器。但它们承载的风险不同——而 148k 对 382k 的 star 数,完全不会告诉你你引入的是哪一种风险。

## 零刷量——以及为什么这仍然值得一说

全部 96 个人类账号中,达到刷量阈值(内部垃圾分 ≥ 3)的数量:**零**,存量分数和当前引擎重算都是零。风险信号命中 19/92(20.7%),但构成全部是**缺失型**或高拒绝率:`mostly_forks`(15)、`no_original_work`(10)、`high_pr_rejection`(4)——**没有**模板化 PR 灌水,**没有**微小 PR 刷量。作为对照,连 Dify 的前 100 里都有 2 个刷量模式账号,基线比例是 0.58%。

一句诚实的补充:按 commit 数排序的前 100 对刷量者是极不友好的栖息地——他们的典型形态是每个仓库塞一两个微小 PR,而 OpenClaw 第 100 名贡献者有 24 个 commit。如果存在刷量,它藏在 368 名贡献者的长尾以及之外的 [2,800 多个匿名邮箱身份](https://github.com/openclaw/openclaw/graphs/contributors)里,本研究不覆盖那一段。这个结果排除的是一件更微妙的事:OpenClaw 的火箭数字是不是一朵人造贡献者云托起来的。不是。这个项目的头部经得起逐个验证。

## 新人浪潮

账号年龄分布是 OpenClaw 与 Dify 彻底分道扬镳的地方:

| | OpenClaw | Dify |
|---|---|---|
| 账号 < 1 年 | **19.6%** | 4.7% |
| 账号 < 2 年 | **26.1%** | 9.3% |
| 账号年龄中位数 | 8.7 年 | 9.0 年 |

分布是双峰的:一个老兵核心(中位 2017 年就在 GitHub 上),加上五分之一 ChatGPT 两周岁时还不存在的账号。这些人是被 AI Agent 时刻卷进开源的——而且他们不是游客。最突出的例子是 [shakkernerd](/u/shakkernerd):2.3 年的账号、362 个 followers,坐在年度最大仓库的 commit 榜第 3 位,94.1 分。而 9.4% 的低分尾巴是同一股浪潮的另一面:全新账号的开源初体验就是给 OpenClaw 修一个小东西。一年之后,他们要么长出履历,要么归于沉寂;到时候我们会重跑这组数据。

## 方法与局限

分数来自 ghfind 的确定性规则——六个维度、只用公开 GitHub 数据、不调用模型,以 AGPL 开源。完整规则与阈值见[方法论](/methodology);每张表背后的聚合数据见 [data.json](/blog/who-builds-openclaw/data.json)。

- **前 100(按 commit 数)是头部,不是整个社区。** OpenClaw 有 368 名代码贡献者,外加约 2,800 个匿名邮箱身份;刷量若存在,会集中在本研究不覆盖的长尾。
- **原始 commit 数依赖工作流。** OpenClaw 的直接提交、agent 驱动风格,单位工作量产生的 commit 比 Dify 这类 squash-merge 项目多;集中度比例在仓库内部可比,跨仓库的 commit 总量不可比。
- **时点数据。** 采集于 2026-07-11,而这个项目移动得比历史上任何项目都快。实时数据见 [OpenClaw 项目页](/developers/repo/openclaw/openclaw)。
- **刷量数据只做聚合披露。** 点名只用于正面语境,这是本站的固定政策。
