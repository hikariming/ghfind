**核心结论**(数据采集于 2026-07-11,对象为 [openclaw/openclaw](https://github.com/openclaw/openclaw) commit 数前 100 的贡献者,使用开源的 [ghfind 引擎](/methodology)评分):

- **GitHub 历史上涨星最快的仓库,按 commit 数算,大部分是一个人写的。** 创始人 [steipete](/u/steipete) 一人贡献了样本 58,487 个 commit 中的 33,482 个——**57.2%**;前三名合计 81.5%,前十名合计 90.2%。
- **没有发现任何刷出来的账号。** 96 个人类贡献者里,没有一个触发引擎的"人造贡献"判定——而即使在我们精心过滤过的 18,947 账号基线里,这个比例也有 0.58%。热度是天文数字,热度背后的人是真的。
- **AI Agent 热潮正在把新人带进开源。** 前 100 贡献者里,19.6% 的 GitHub 账号注册不满一年(作为对照,名气相当但更年长的 Dify 只有 4.7%)。新人里已经出了高手:commit 数第 3 的贡献者,账号只有 2.3 年,拿了 94.1 分。

## 为什么研究 OpenClaw

[OpenClaw](https://github.com/openclaw/openclaw) 不需要介绍:2025 年 11 月 24 日由 Peter Steinberger([steipete](/u/steipete),PSPDFKit 创始人)创建,2026 年 1 月一周之内改了两次名(Clawdbot → Moltbot → OpenClaw),是史上最快达到 10 万 star 的项目。截至 2026-07-11,它有 **382,580 个 star、80,292 个 fork、368 名代码贡献者**——距第一个 commit 只过了七个半月。

这样的涨星曲线,正是我们做这个引擎的原因:当一个数字涨得这么快,它下面到底有没有真东西?我们取 commit 数前 100 的贡献者,剔除 4 个 bot(包括项目自己的 `clawsweeper` 和 `openclaw-clownfish`),对全部 96 个人类账号评分,其中 92 个留有完整原始指标快照。

## 贡献者质量:榜单头部非常能打

| | OpenClaw 前 100 | Dify 前 100 | 19k 基线 |
|---|---|---|---|
| 中位分 | **79.7** | 78.2 | 42.5 |
| ≥ 90 分(夯) | **21.9%** | 15.6% | 3.7% |
| ≥ 70 分(可信) | **69.8%** | 70.8% | 20.1% |
| < 40 分(低价值) | **9.4%** | 5.2% | 48.6% |

(Dify 列来自[同系列研究](/blog/who-builds-dify),同一周、同一个引擎。)

OpenClaw 的头部贡献者,每五个就有一个拿到 90 分以上——放眼整个 GitHub,大约二十七个账号里才有一个能做到。当全行业的目光都聚在一个项目上,最好的开发者也会跑来给它写代码。但另一头也要看:9.4% 的人低于 40 分,几乎是 Dify 的两倍。这么大的关注度,同样会吸来一批主页上几乎什么都没有的全新账号——他们是从哪来的,下面讲账号年龄的部分会解释。

## 一双手

前 100 样本的 commit 总数:58,487——是 Dify(8,434)的七倍,而用的日历时间只有五分之一。

| commit 占比 | OpenClaw | Dify |
|---|---|---|
| 最活跃的 1 人 | **57.2%** | 8.4% |
| 前 3 名 | **81.5%** | 21.0% |
| 前 5 名 | **86.0%** | 31.3% |
| 前 10 名 | **90.2%** | 49.8% |

[steipete](/u/steipete) 的 33,482 个 commit 摊到 229 天里,平均**每天 146 个**。没有人能打字打这么快——但一个指挥着一群编程 agent、负责审核和合并它们产出的人,可以合并得这么快,而 OpenClaw 出了名就是这么造出来的。引擎给这个账号打了 **100 分满分**:17 年的 GitHub 历史、52,067 个 followers、2,772 个合并 PR——离"假账号"要多远有多远。产出是真的,只是它集中在一双手上,集中到这个体量的项目里前所未见的程度。

第二梯队人不多但都过硬:[vincentkoc](/u/vincentkoc)(10,502 个 commit,96.5 分)、[shakkernerd](/u/shakkernerd)(3,688 个,94.1 分)、[obviyus](/u/obviyus)(1,771 个,93.2 分)。第 10 名往后,没有任何人的 commit 占比达到 0.5%。

两种干法都能成:Dify 靠一个真正广泛的社区一起写;OpenClaw 靠一个人拍所有的板,跑出了历史上没有过的速度。但两者的风险不一样——那个人一停,一切都停——而 148k 对 382k 的 star 数,完全不会告诉你你选的是哪种风险。

## 没有刷出来的账号——为什么这仍然值得一说

全部 96 个人类账号里,触发"人造贡献"判定的数量:**零**。用库里的存量分数是零,用当前版本引擎重算还是零。警示信号出现在 19/92 个账号上(20.7%),但全部属于"主页单薄"或"PR 被拒得多"这一类:`mostly_forks`(15 个)、`no_original_work`(10 个)、`high_pr_rejection`(4 个)。没有一个账号在批量生产 PR 标题,没有一个靠琐碎 PR 堆履历。作为对照,连 Dify 的前 100 里都有 2 个这样的账号,基线比例是 0.58%。

诚实地补一句:按 commit 数排前 100,本身就会把刷号的人挡在门外——他们的惯用手法是给每个仓库塞一两个改动极小的 PR,而 OpenClaw 第 100 名贡献者有 24 个 commit,靠改错别字是进不来的。如果 OpenClaw 周边真有刷出来的活动,它藏在 368 名贡献者的长尾、以及那之外 [2,800 多个匿名邮箱身份](https://github.com/openclaw/openclaw/graphs/contributors)里——本研究没有覆盖那一段。但这个结果排除了一项更严重的怀疑:OpenClaw 的惊人数字,是不是一群批量注册的假账号撑起来的。不是。这个项目最核心的这批人,一个一个查,都经得起查。

## 新人潮

账号年龄,是 OpenClaw 和 Dify 彻底不像的地方:

| | OpenClaw | Dify |
|---|---|---|
| 账号 < 1 年 | **19.6%** | 4.7% |
| 账号 < 2 年 | **26.1%** | 9.3% |
| 账号年龄中位数 | 8.7 年 | 9.0 年 |

贡献者明显分成两拨:一拨是 2017 年前后就上 GitHub 的老手,构成项目的核心;另有五分之一,账号一年前几乎还不存在。这批新人是被 AI Agent 热潮带进开源的——而且他们不是来打个卡就走的。最亮眼的例子是 [shakkernerd](/u/shakkernerd):账号 2.3 年、362 个 followers,坐在年度最大仓库的 commit 榜第 3 位,94.1 分。而那 9.4% 的低分账号,是同一股潮水的另一面:全新的账号,人生第一次参与开源就是给 OpenClaw 修了个小东西。一年之后,他们要么攒出真履历,要么销声匿迹——到时候我们把数据重跑一遍就知道了。

## 方法与局限

分数来自 ghfind 的确定性规则——六个维度、只用公开 GitHub 数据、不调用模型,以 AGPL 开源。完整规则与阈值见[方法论](/methodology);每张表背后的汇总数据见 [data.json](/blog/who-builds-openclaw/data.json)。

- **前 100(按 commit 数)是项目的头部,不是整个社区。** OpenClaw 有 368 名代码贡献者,外加约 2,800 个匿名邮箱身份;刷出来的活动如果存在,会藏在我们没有评分的那条长尾里。
- **原始 commit 数和工作流有关。** OpenClaw 这种 agent 驱动、直接提交的干法,同样的工作量会产生比 Dify 这类"squash 合并"项目多得多的 commit。所以占比在同一个仓库内有意义,拿 commit 总数跨仓库比较没有意义。
- **这是一张快照。** 数据采集于 2026-07-11,而这个项目跑得比历史上任何项目都快。实时数据见 [OpenClaw 项目页](/developers/repo/openclaw/openclaw)。
- **刷量相关的发现只公布汇总数字。** 点名只在说好话的时候——这是本站的固定规矩。
